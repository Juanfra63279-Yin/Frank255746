Modernización La Moderna

1. Introducción

Proyecto: Modernización Digital de Vinatería La Moderna.

Objetivo: Migración de un sistema monolítico local hacia un ecosistema web escalable en la nube.

Cliente: Luis Ramírez.

2. Problemática y Justificación

Problemática:

  • Ventas: Carencia de e-commerce; mercado limitado.
  • Inventario: Conteo manual con alta tasa de error.
  • Finanzas: Cierres dependientes de tickets físicos.
  • Seguridad: Gestión de accesos no controlada.

Justificación:

  • Captación de clientes mediante reservas web.
  • Automatización directa del Punto de Venta (POS).
  • Escalabilidad y auditoría de datos en la nube.

3. Puntos de mejora (Requerimientos)

  • Catálogo de Reservas: Portal interactivo 24/7 para apartados y optimización en mostrador.
  • Motor de Recomendación: Algoritmos basados en grafos para sugerir sustitutos en stock.
  • Gestión de Roles: Segregación estricta de permisos (Administrador, Empleado, Cliente).
  • Dashboard Centralizado: Control integrado de inventario y proveedores.

4. Arquitectura

  • Frontend: HTML5 + CSS3 (Estructurado y responsivo).
  • Backend: PHP (Patrón Modelo-Vista-Controlador).
  • Persistencia: MySQL (Integridad referencial).
  • Infraestructura: Cloud Web Hosting.

5. Grafo

Definición: Motor de recomendación visual (nodos y aristas) para sugerencias de sustitución.

  • Cervezas: Corona ↔ Modelo (0.85).
  • Tequilas: Don Julio 70 ↔ Maestro Dobel (0.70).
  • Vodkas: Smirnoff ↔ Absolut (0.75).
  • Rones: Bacardí Blanco ↔ Añejo (0.60).

6. Conclusión y Trabajo a corto plazo

Conclusión: La transición erradica la pérdida de registros, otorga ventaja competitiva sin altos costos de infraestructura y centraliza las decisiones empresariales.

Corto Plazo (Fase 2): Migración a arquitectura React. Desacoplamiento de lógica e interfaz (SPA) y preparación para consumo de APIs.

7. Escalabilidad

  • App Móvil (Fase 3): Versión iOS/Android nativa con notificaciones push.
  • IA Avanzada (Fase 4): Machine Learning para perfiles de recomendación.
  • Multi-Sucursal (Fase 5): Arquitectura multi-tenant para despliegue en franquicias.
