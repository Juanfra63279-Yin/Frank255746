01 — Introducción

Vinatería La Moderna:
El Reto de la Modernización

El equipo InnovateX fue contratado por Luis Ramírez, propietario de Vinatería La Moderna, para migrar su sistema operativo local a un ecosistema web completo, escalable y accesible desde la nube.

Proyecto

Modernización Digital de Vinatería La Moderna — migración de arquitectura de escritorio local hacia plataforma web en la nube con PHP, HTML/CSS y MySQL bajo patrón MVC.

Cliente

Luis Ramírez — Propietario y Administrador de Vinatería La Moderna. Necesidad crítica: acceso remoto, catálogo digital y control de inventario en tiempo real.

Equipo Consultor

InnovateX — 9 ingenieros de La Salle Nezahualcóyotl. Metodología: SCRUM Framework con 4 sprints estructurados, sincronización semanal y entrega iterativa.

Sistema Origen → Destino

NetBeans 8.2 + Java Swing + XAMPP (local) → HTML5 + CSS3 + PHP (MVC) + MySQL desplegado en Cloud Web Hosting con disponibilidad 24/7.

Sección 02

Brechas Operativas y Justificación

La dependencia de procesos locales ralentizaba el crecimiento comercial. A continuación se presentan los dolores críticos detectados en el análisis del negocio de licores y las soluciones integradas:

Canal de Ventas Limitado

Ausencia total de un canal digital, forzando transacciones presenciales y perdiendo posicionamiento frente al mercado online actual.

Solución: Catálogo web 24/7 interactivo para reserva directa.

Inventariado Ineficiente

Conteos de stock manuales propensos a errores en la categorización de licores finos, sin actualización inmediata de existencias.

Solución: Base de datos relacional con sincronización inmediata.

Invisibilidad Financiera

Los cortes de caja y cierres diarios requerían la validación tediosa de tickets físicos impresos, impidiendo auditorías ágiles.

Solución: Exportación analítica nativa en formato CSV.
Sección 03

Puntos de Mejora Técnicos

  • Catálogo y Apartado Web Interactivos Permite reservas ininterrumpidas agilizando la entrega posterior en mostrador, liberando la infraestructura de pasarelas de pago complejas.
  • Seguridad de la Información Bases de datos robustas en la nube reemplazan las bitácoras locales, previniendo pérdidas de registros irreemplazables.
  • Control de Accesos por Privilegios Segregación granular de roles (Administrador, Empleado, Cliente) protegiendo la información sensible del negocio.

Impacto Operativo

Centralización absoluta de los registros, erradicando pérdidas documentales previas.

04 — Arquitectura

Arquitectura General
del Sistema

El sistema abandona el esquema local obsoleto para implementar una arquitectura MVC con tres capas tecnológicas bien definidas desplegadas en la nube.

Frontend

Interfaces reactivas y diseño visual estructurado en HTML5 y CSS3 puro para la mejor experiencia de usuario al navegar el catálogo de reservas.

HTML5 · CSS3

Backend

Procesamiento y control centralizado en PHP bajo patrón MVC para gestión de usuarios, inventarios y ciclo completo de reservaciones.

PHP · MVC

Base de Datos

Esquema relacional centralizado con integridad referencial en el registro de productos, proveedores, reservaciones e históricos auditables.

MySQL · Relacional

Infraestructura

Despliegue en servidor web externo (InfinityFree) que supera las barreras físicas del antiguo sistema local, accesible 24/7 desde cualquier dispositivo.

Cloud · Web Hosting

Control de Accesos

Segregación de roles con sesiones seguras PHP para administrador, empleado y cliente. Módulo de asignación granular de privilegios.

PHP Sessions · RBAC

Grafos de Productos

Motor de recomendaciones basado en grafos de sustitución (matrices de adyacencia ponderadas) que optimiza la experiencia al reservar.

Grafos · Wolfram

Sección 06

Conclusión y Trabajo Futuro

La transición ágil erradica las ineficiencias de control local, confiriendo una ventaja competitiva medible en el sector sin la necesidad de costos pesados de infraestructura para Vinatería La Moderna.

Trabajo Futuro — Escalabilidad

Hoja de Ruta
del Sistema

El sistema actual establece la base para una evolución progresiva hacia capacidades de mayor complejidad y alcance comercial.

Transición de entorno estático hacia ecosistema de componentes dinámicos (SPA). Desacoplamiento de lógica e interfaz mediante programación orientada a aspectos, optimizando rendimiento y preparando para consumo de APIs.

Fase 4 — Largo Plazo

IA para Recomendaciones Avanzadas

Evolución del motor de grafos hacia Machine Learning que aprenda del historial de reservas para personalizar sugerencias por perfil de cliente.

Fase 5 — Escalabilidad

Arquitectura Multi-Sucursal

Arquitectura multi-tenant para gestionar múltiples sucursales de La Moderna desde un único panel de administración centralizado en la nube.

Frontend: Detalles Técnicos

02
Backend: Detalles Técnicos

Se rige bajo la Arquitectura Modelo-Vista-Controlador (MVC) en PHP, separando la lógica de negocio de la interfaz gráfica. Garantiza el procesamiento seguro de formularios y el ciclo completo de validación para las reservaciones.

×
03
Base de Datos: Estructura

Construida sobre MySQL con integridad referencial mediante llaves primarias y foráneas. El modelo relacional optimiza las consultas de inventario cruzadas y mantiene la trazabilidad absoluta de movimientos de stock, cortes de caja y auditorías históricas.

×
04
Infraestructura en la Nube

Transición definitiva de localhost (XAMPP) a un Servidor Web Cloud (InfinityFree). Elimina las barreras físicas de la sucursal, con disponibilidad 24/7 y flexibilidad para escalar el ecosistema.

×
05
Control de Accesos

Sistema de Autenticación basado en Sesiones PHP Seguras con encriptación de credenciales. Segregación estricta de Roles: Administrador, Empleado y Cliente; blindando módulos críticos financieros.

×
06
Grafos de Productos

Lógica algorítmica apoyada en Teoría de Grafos con matrices de adyacencia ponderadas en Wolfram. Los nodos son productos y las aristas definen los grados de similitud. Motor de recomendaciones: si un licor se agota, el sistema sugiere sustitutos directos de forma eficiente.

Modernización Ágil

La implementación con stack ligero (PHP, HTML, CSS) garantiza que La Moderna migre de su sistema NetBeans 8.2 obsoleto hacia un ecosistema web sin necesidad de invertir en infraestructura costosa. La metodología SCRUM con 4 sprints garantizó coherencia estricta con las reglas de negocio.

Nuevo Canal de Contacto

Al limitar el sistema al apartado físico y ejecutar el cierre financiero en mostrador, se evita la exposición a fraudes por pagos en línea mientras se expande el alcance comercial.
